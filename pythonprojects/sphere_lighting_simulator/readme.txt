This is a mathematical model of lighting on a sphere, the only real difference between the two files is that sphere_graphics has an UI and sphere_basic runs in the command window. 

The program works by creating a grid of pixels and essentially virtually assigning to each one an elevation to mimic a sphere sitting on a plane. When simulating the lighting of the sphere it then uses the xy and elevation derived from the xy coordinates of the pixel as well as the directional vector of the incoming light to estimate the brightness via their dot product. 

The shadow on the ground is some very rudimentary raytracing checking whether a virtual line drawn from the ground up towards the light source intersects with the sphere. It is quite computation heavy as the shadow was a later add on to the project and the structure was not built for it, but it has been somewhat optimized in the sense it calculates the lower and upper bound of where the shadow could be first and then only checks the pixels inside this area. 

The structure is such that the image is a distinct class that holds all the data relevant such as radius, image data, width, height ect. The array of pixels actually only hold one single piece of data which is brightness, things like x and y coordinates or elevation are all calculated using the index of the pixel and initial conditions such as radius and then fed directly into the light function. Inside are also all the functions and essentially the program creates one instance in the beginning and as you simulate the light you update the pixel array of this instance. There is then also a class that manages the interface and calling on the appropriate functions as you click to simulate the light.
